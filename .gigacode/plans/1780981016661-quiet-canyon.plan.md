## Функциональность

### Микросервис 1: credit-application-service (Оформление заявки)
- **REST API**:
  - `POST /api/v1/credit-applications` - создание заявки на кредит
  - `GET /api/v1/credit-applications/{id}` - получение статуса заявки
  
- **Функции**:
  - Принимает заявку от пользователя
  - Сохраняет заявку в БД со статусом "В ОБРАБОТКЕ"
  - Отправляет событие в Kafka для обработки
  - Получает ответ от микросервиса обработки через RabbitMQ
  - Обновляет статус заявки в БД при получении ответа

### Микросервис 2: credit-processor-service (Обработка заявки)
- **Функции**:
  - Получает заявку из Kafka
  - Проверяет условие: платеж по кредитам ≤ 50% от доходов
  - Отправляет решение (ОДОБРЕНО/ОТКАЗАНО) в RabbitMQ

## Технологии
- **Spring Boot 4.0.6** + Java 21
- **H2 Database** (для разработки)
- **Kafka** (для обмена заявками от сервиса 1 к сервису 2)
- **RabbitMQ** (для ответов от сервиса 2 к сервису 1)
- **Flyway** (для миграций БД)
- **Docker & Docker Compose** для развертывания инфраструктуры

## Порядок реализации

1. **Создать родительский pom.xml** с двумя модулями
2. **Создать структуру credit-application-service**:
   - pom.xml с зависимостями (Web, JPA, H2, Kafka, RabbitMQ, Flyway)
   - Entity, Repository, Controller, DTO
   - Конфигурация Kafka и RabbitMQ
   - Consumer для получения ответов
   - Producer для отправки в Kafka
3. **Создать структуру credit-processor-service**:
   - pom.xml с зависимостями (Kafka, RabbitMQ)
   - Consumer для получения из Kafka
   - Producer для отправки в RabbitMQ
   - Processor для принятия решений
4. **Создать docker-compose.yml** с Kafka, RabbitMQ и контейнерами
5. **Создать broker-messages.http** для тестирования
6. **Создать миграции Flyway** для БД

## Команды для запуска
```bash
# Запуск инфраструктуры
docker-compose up -d

# Сборка и запуск сервисов
mvn clean install
mvn spring-boot:run -pl credit-application-service
mvn spring-boot:run -pl credit-processor-service
```
